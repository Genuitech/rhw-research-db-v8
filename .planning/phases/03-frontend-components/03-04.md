# Plan 03-04: EntryViewer & RedlineEditor

## Objective

Build the component for viewing approved entries with redline (diff) editing capabilities.

## Must-Haves

- [ ] EntryViewer displays entry (title, client, question, memo, tags, status)
- [ ] Approval history shows all actions with timestamps
- [ ] Edit button triggers RedlineEditor
- [ ] RedlineEditor shows original vs. proposed changes
- [ ] Diff highlighting (additions in green, deletions in red)
- [ ] Submit edited version for re-approval
- [ ] Breadcrumb/back navigation
- [ ] Component tests passing
- [ ] Helper tests passing

## Tasks

### 1. Create EntryViewer.jsx

- Display full entry details:
  - Title (type + client)
  - Question/Topic
  - Memo (original text)
  - Tags
  - Status (approved/pending/rejected)
  - Approval history (who, when, action)
- Edit button (only for non-final statuses)
- Liquid glass design
- Breadcrumb or back navigation

### 2. Create RedlineEditor.jsx

- Side-by-side or inline diff view
- Original text on left, proposed on right
- Diff highlighting:
  - Green for additions
  - Red for deletions
  - Gray for unchanged
- Edit textarea for user to make changes
- Submit button (sends to `/api/submitEntry` for re-approval)
- Cancel button (discard changes)

### 3. Create helper: redlineUtil.js

- calculateDiff(original, edited) — Detect additions/deletions
- formatRedlineHTML(diff) — Render with highlighting
- Use diff-match-patch library (or simple char-by-char comparison)

### 4. Create tests

- EntryViewer.test.jsx: Rendering, data display, edit button
- RedlineEditor.test.jsx: Diff calculation, edit submission
- redlineUtil.test.js: Diff logic

### 5. Verify integration

- Entry data loads correctly
- Approval history displays
- Diff calculation accurate
- Edit submission works
- Redirects back to search after submission

## Files to Create

- `frontend/src/components/EntryViewer.jsx`
- `frontend/src/components/RedlineEditor.jsx`
- `frontend/src/utils/redlineUtil.js`
- `frontend/tests/EntryViewer.test.jsx`
- `frontend/tests/RedlineEditor.test.jsx`
- `frontend/tests/redlineUtil.test.js`

## Dependencies

- Optional: `diff-match-patch` npm package for advanced diff logic
- Or implement simple diff using string comparison

## Success Criteria

- [ ] EntryViewer renders and displays entry correctly
- [ ] Approval history shows all actions
- [ ] Edit button opens RedlineEditor
- [ ] Diff calculation is accurate
- [ ] Diff highlighting is clear (additions, deletions)
- [ ] Edit submission works
- [ ] User redirected back to search after submission
- [ ] All component tests pass
- [ ] All helper tests pass
- [ ] Responsive design verified
- [ ] No console errors
