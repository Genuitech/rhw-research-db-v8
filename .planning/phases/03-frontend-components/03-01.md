# Plan 03-01: SearchInterface Component

## Objective

Build the main search UI with filters, pagination, and result display using liquid glass design and backend API integration.

## Must-Haves

- [ ] Search input with debounced queries (300ms delay)
- [ ] Filter sidebar: status (approved, pending, rejected), type (memo, sop, policy), tags
- [ ] Results grid with pagination (20 items/page)
- [ ] Empty state, loading state, error state
- [ ] Responsive design (mobile: 1 col, tablet: 2, desktop: 3)
- [ ] Integration with `/api/search` endpoint
- [ ] Component tests passing
- [ ] Hook tests passing

## Tasks

### 1. Create SearchInterface.jsx

- Input field with debounced search (300ms)
- Filter sidebar with checkboxes/dropdowns
- Results grid with entry cards
- Pagination controls (prev/next, page indicator)
- Responsive layout using grid
- Empty/loading/error states
- Uses liquid glass classes from globals.css

### 2. Create useSearch.js hook

- State: query, filters, page, results, total, loading, error
- API call to `/api/search?q={query}&status={status}&type={type}&tags={tags}&page={page}&limit=20`
- Debounce search input (300ms)
- Handle pagination
- Error handling

### 3. Create tests

- SearchInterface.test.jsx: Rendering, filter interaction, pagination
- useSearch.test.js: API calls, debouncing, state updates

### 4. Verify integration

- API calls use correct endpoint
- Results formatted correctly
- Pagination works
- Responsive on all viewport sizes

## Files to Create

- `frontend/src/components/SearchInterface.jsx`
- `frontend/src/hooks/useSearch.js`
- `frontend/tests/SearchInterface.test.jsx`
- `frontend/tests/useSearch.test.js`

## Success Criteria

- [ ] Component renders without errors
- [ ] Search queries work and return results
- [ ] Filters work (status, type, tags)
- [ ] Pagination works (next/prev)
- [ ] Empty/loading/error states display
- [ ] Responsive design verified (mobile/tablet/desktop)
- [ ] All component tests pass
- [ ] All hook tests pass
- [ ] No console warnings or errors
