# Plan 03-02: SubmissionForm Component

## Objective

Build the data entry form for submitting new research entries with validation and backend integration.

## Must-Haves

- [ ] Form fields: type, client, question, memo, tags, conversation (optional)
- [ ] Input validation: required fields, min 20 chars for memo
- [ ] Success/error feedback
- [ ] Submit button disabled during request
- [ ] Integration with `/api/submitEntry` endpoint
- [ ] Liquid glass design
- [ ] Form tests passing
- [ ] Hook tests passing

## Tasks

### 1. Create SubmissionForm.jsx

- Form fields:
  - Type (select: memo, sop, policy)
  - Client name (text input, required)
  - Question/Topic (textarea, required, min 20 chars)
  - Memo/Answer (textarea, required, min 20 chars)
  - Tags (comma-separated input or multi-select)
  - Conversation transcript (optional, textarea)
- Form validation with error messages
- Submit button (disabled during request)
- Success message on submission
- Error handling with retry
- Liquid glass styling

### 2. Create useEntries.js hook

- submitEntry(formData) — POST to `/api/submitEntry`
- getEntry(id) — GET from `/api/entry/{id}`
- State: currentEntry, loading, error, successMessage
- Error handling and recovery

### 3. Create tests

- SubmissionForm.test.jsx: Rendering, validation, submission
- useEntries.test.js: API calls, state management

### 4. Verify integration

- API calls use correct endpoint
- Validation prevents invalid submissions
- Success feedback works
- Error handling works

## Files to Create

- `frontend/src/components/SubmissionForm.jsx`
- `frontend/src/hooks/useEntries.js`
- `frontend/tests/SubmissionForm.test.jsx`
- `frontend/tests/useEntries.test.js`

## Success Criteria

- [ ] Form renders without errors
- [ ] All fields render correctly
- [ ] Validation prevents submission of invalid data
- [ ] Submit button sends correct API payload
- [ ] Success message displays after submission
- [ ] Error handling works (retry, error message)
- [ ] All component tests pass
- [ ] All hook tests pass
- [ ] Form clears after successful submission
- [ ] Responsive design verified
