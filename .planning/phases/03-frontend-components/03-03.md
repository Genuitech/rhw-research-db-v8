# Plan 03-03: ApprovalDashboard Component

## Objective

Build the admin dashboard for reviewing pending entries and approving/rejecting them.

## Must-Haves

- [ ] Display list of pending entries
- [ ] Entry card preview: title, client, question, author, submitted date
- [ ] Preview pane: full memo + conversation (admin only)
- [ ] Action buttons: Approve, Reject, Request Changes
- [ ] Quick filters: status, time range
- [ ] List refreshes after approval
- [ ] Admin-only access (hide conversation from staff)
- [ ] Component tests passing
- [ ] Hook tests passing

## Tasks

### 1. Create ApprovalDashboard.jsx

- Pending entries list view
- Entry cards with summary info
- Preview pane (right sidebar)
- Action buttons (Approve, Reject, Request Changes)
- Quick filters (status, date range)
- Responsive layout (desktop: side-by-side, mobile: stacked)
- Liquid glass design
- Admin role check (show conversation only to admins)

### 2. Create useApproval.js hook

- getPendingEntries() — GET `/api/search?status=pending`
- approveEntry(id, action) — POST `/api/approveEntry`
- State: pendingEntries, selectedEntry, loading, error
- Refresh list after approval
- Error handling

### 3. Create tests

- ApprovalDashboard.test.jsx: Rendering, entry selection, approval actions
- useApproval.test.js: API calls, state management, filtering

### 4. Verify integration

- API calls correct
- List loads pending entries
- Approval actions work
- List refreshes after action
- Admin-only content hidden from non-admins

## Files to Create

- `frontend/src/components/ApprovalDashboard.jsx`
- `frontend/src/hooks/useApproval.js`
- `frontend/tests/ApprovalDashboard.test.jsx`
- `frontend/tests/useApproval.test.js`

## Success Criteria

- [ ] Dashboard loads and displays pending entries
- [ ] Entry preview works (click to expand)
- [ ] Approve/Reject/Request Changes buttons work
- [ ] List refreshes after approval action
- [ ] Admin can see full conversation
- [ ] Staff role restricted from viewing conversation
- [ ] All component tests pass
- [ ] All hook tests pass
- [ ] Error handling works (network errors, invalid actions)
- [ ] Responsive design verified
